## Challenge ‚Äî Try SOLID literally, then make peace with it

The point of today is to over-engineer the design by blindly following SOLID to the letter. Pick a small hotspot in your code.

Then pause, write down your observation, complexity metrics, pain points, etc.

Then, 'think' SOLID and go for a more pragmatic design.

The end result does **not** need to be perfectly SOLID.  
It just needs to be **clearer and more intentional** than what you started with.

If you still have the complexity tool from Day 1, you can optionally compare the original and your final version to see whether complexity went up, down, or just moved.

Share your code and observations along the way.
